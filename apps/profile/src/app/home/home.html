✨ NX Monorepo / Multi App ✨

Profile is an Angular 21 monorepo with reactive modeling

Libraries I use to explore and validate new technologies. Being experimental, you might also find @todos or fake integrations :)

Tech Stack: Angular 21 | RxJS | NgRx Signals | Material Design | Tailwind CSS

Architecture: Nx monorepo with shared libraries | Domain-driven design | Feature modules | Lazy loading

Backend: NestJS API | MongoDB | REST endpoints

  • Users API Service -> CRUD operations with typed HTTP client (endpoints documented in Swagger)

  • User dialog edition -> Lab Integration / Dialog for inline user editing

⭐ Auth Library

Centralized authentication and authorization with token validation, guards and role-based access control

  • Feature library -> ValidateUserModel singleton service that validates access tokens, refresh tokens and expiration
  • Route Guards -> CanActivate guards to protect routes based on roles and permissions
  • HTTP Auth Interceptor -> Automatically injects Bearer tokens in Authorization headers and handles token refresh
  • Infrastructure library -> Interfaces and types: AuthToken, UserCredentials, AuthResponse, PermissionModel
  • JWT Decoder -> Utility to decode and validate JWT token structure
  • Role-based Access Control (RBAC) -> Permission validation at component and service level

⭐ Lab Libraries

  • Buttons | UI lib
    • LabMiniFabButton -> MatMiniFabButton wrapper with extended properties (color, disabled, loading state)
    • LabButton -> MatButton wrapper with ripple effects, themes and accessible aria labels
    • Standalone Components -> Components without modules for simplified imports
  • Dialog | Feature lib
    • DialogService -> MatDialog wrapper with generic types for passed data and results
    • ConfirmDialog -> Reusable dialog for confirmations with customizable buttons
    • DialogConfig -> Centralized configuration of width, position, backdrop, animations
  • Schematics | Feature Schematics
    • FeatureRouteModule -> Generates a FeatureModule with Routes, components and lazy loading configured
    • Component Schematic -> Creates standalone components with selector, template and structured styles
    • Service Schematic -> Generates services with providedIn: 'root' and generic types
  • Utilities & Helpers
    • Type Guards -> Functions for type narrowing and runtime type checking
    • Pipe Utilities -> Custom pipes for data transformation (date formatting, currency, etc)
    • RxJS Operators -> Custom operators for common patterns (shareReplay, takeUntilDestroyed)

📐 Architecture & Patterns

Folder Structure

  • libs/users/feature - UI and feature logic
  • libs/users/+state - NgRx signals and reducer
  • libs/users/data-access - API services
  • libs/users/ui - Presentational components
  • libs/auth/infrastructure - Types and interfaces

Design Patterns

  • Smart/Dumb Components
  • Observable Data Services
  • Singleton Services (Auth, API)
  • Reactive Forms Pattern
  • Error Boundary Components

Performance Optimizations: OnPush change detection | Lazy-loaded modules | RxJS shareReplay caching | @defer for deferred rendering with placeholders | Tree-shaking compatible

🛠️ Backend: NestJS & MongoDB

REST API built with NestJS and data persistence in MongoDB using Mongoose ODM

  • NestJS Controllers -> REST routes for user CRUD, authentication and input validation
  • MongoDB + Mongoose -> Typed schemas, schema-level validation, indexing for optimized queries
  • JWT Authentication -> Passport JWT strategy for token validation on protected endpoints
  • DTOs (Data Transfer Objects) -> Validation with class-validator for request payloads
  • Middleware & Guards -> AuthGuard and ValidationPipe for security and data transformation
  • Error Handling & Security -> Centralized exception filters, CORS, helmet headers, rate limiting
  • OpenAPI/Swagger -> Automatic endpoint documentation at /api/docs
