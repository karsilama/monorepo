⭐ Users Library

Complete integration with Users REST API using reactive signals and modern Angular patterns

Key Features

  • REST API Integration - Typed HTTP client with CRUD operations
  • Reactive Data Loading - Angular signals for state management
  • Smart/Dumb Components - Separated UI and business logic
  • HTTP Interceptors - Token handling & caching
  • Reactive Forms - Real-time validation with custom validators

Architecture

libs/users/feature - UI and feature logic

libs/users/+state - NgRx signals & reducers

libs/users/data-access - API services

libs/users/ui - Presentational components

Implementation

 ⭐ User Domain Model

Complete user data model with comprehensive properties for managing user information across the platform

Core Properties

id - number - Unique identifier

firstName - string - First name

lastName - string - Last name

email - string - Email address

username - string - Username

password - string - Password hash

phone - string - Phone number

role - 'admin' | 'moderator' | 'user'

Personal Info

age - number - Age in years

birthDate - string - Birth date

gender - 'male' | 'female'

maidenName - string - Maiden name

image - string - Profile image URL

bloodGroup - string - Blood group

height - number - Height in cm

weight - number - Weight in kg

Physical & Medical

eyeColor - string - Eye color

hair - Hair object

├─ color: string

└─ type: string

Location & Network

address - Address object

├─ address, city, state, country

├─ coordinates (lat, lng)

└─ postalCode

ip - string - IP address

macAddress - string - MAC address

userAgent - string - User agent string

Professional & Financial

company - Company object

├─ name, department, title

└─ address

university - string - University name

bank - Bank object

├─ cardNumber, cardType

├─ cardExpire, currency

└─ iban

Legal & Crypto

ssn - string - Social Security Number

ein - string - Employer ID Number

crypto - Crypto object

├─ coin (bitcoin, ethereum...)

├─ wallet address

└─ network

Usage Notes

  • • All properties are optional (?)
  • • Nested objects allow hierarchical data organization
  • • Used throughout Users library (state, services, components)
  • • Forms filter with UserFormModel (firstName, lastName, email only)
  • • Filtering with UserDomainFilteredModel
  • • API response type: UsersResponse (array of UserDomainModel)
